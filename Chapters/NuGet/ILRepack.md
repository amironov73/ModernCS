### Утилита ILRepack

ILRepack -- альтернатива утилите [ILMerge](http://www.microsoft.com/downloads/details.aspx?FamilyID=22914587-B4AD-4EAE-87CF-B14AE6A939B0&displaylang=en)/[Mono.Merge](http://evain.net/blog/articles/2006/11/06/an-introduction-to-mono-merge).

GitHub: https://github.com/gluck/il-repack, NuGet: https://www.nuget.org/packages/ILRepack/

Синтаксис командной строки совместим с ILMerge:

```
Syntax: ILRepack.exe [options] /out:<path> <path_to_primary> [<other_assemblies> ...]

 - /help                выводит данную посказку
 - /keyfile:<path>      задаёт файл ключа для подписи результирующей сборки
 - /keycontainer:<name> задаёт контейнер ключа для подписи результирующей сборки (имеет приоритет над /keyfile)
 - /log:<logfile>       включает логирование (в файл, если указан) (по умолчанию выключено)
 - /ver:M.X.Y.Z         версия результирующей сборки
 - /union               объединяет типы с идентичными именами в один
 - /ndebug              запрещает генерацию символьного файла
 - /copyattrs           копирует атрибуты сборки (по умолчанию берутся атрибуты первичной сборки)
 - /attr:<path>         взять атрибуты указанной сборки
 - /allowMultiple       если задан ключ copyattrs, разрешает множественные атрибуты (если подобное допускается типом)
 - /target:kind         задаёт тип результирующей сборки (библиотека, exe, winexe, по умолчанию берётся тип первичной сборки)
 - /targetplatform:P    задаёт платформу результирующей сборки(поддерживается v1, v1.1, v2, v4)
 - /xmldocs             объединяет также XML-документацию
 - /lib:<path>          задаёт дополнительные директории для поиска сборок (можно указать более одного раза)
 - /internalize         помечает все типы не из первичной сборки 'internal'
 - /delaysign           устанавливает ключ, но не подписывает сборку (отложенная подпись)
 - /usefullpublickeyforreferences - НЕ РЕАЛИЗОВАНО
 - /align               - НЕ РЕАЛИЗОВАНО
 - /closed              - НЕ РЕАЛИЗОВАНО
 - /allowdup:Type       разрешает дублирование указанного типа во входных сборках
 - /allowduplicateresources разрешает дублирование ресурсов во воходных сборках (по умолчанию дубли игнорируются)
 - /zeropekind          разрешает сборки с Zero PeKind (но, очевидно, лишь IL будет объединён в общую сборку)
 - /wildcards           разрешает (и резольвит) шаблоны имени файлов (например, `*`.dll) во входных сборках
 - /parallel            использовать максимально возможное количество процессоров/ядер при слиянии сборок
 - /pause               пауза непосредственно сразу по завершении слияния (для отладки)
 - /verbose             более подробное логирование
 - /out:<path>          путь до результирующей сборки/конфигурационного файла/символьного файла/файла XML-документации
 - <path_to_primary>    первичная сборка, дающая имя и версию результирующей сборке
 - <other_assemblies>   прочие сборки

Примечание: для совместимости все опции могут предваряться любым из префиксов: '/', '-' или '--'.
```
